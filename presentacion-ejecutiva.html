Proyecto Caso Testigo - CI/CD

Integración Continua y despliegue automatizado para un sistema gestor de tareas empresarial.

  • Curso: Ingeniería de Software FESC
  • Unidad: Integración continua y proyecto final
  • Objetivo: Unir backend, frontend, pruebas y automatización en un solo repositorio.

Agenda

  • Arquitectura general
  • Backend FastAPI y pruebas
  • Frontend React + Redux
  • Pipeline CI/CD
  • Métricas y cobertura
  • Riesgos y aprendizajes
  • Roadmap siguiente iteración

Arquitectura

  • Backend FastAPI sobre Python 3.11, contenedor Docker opcional.
  • Base de datos PostgreSQL gestionada con Docker Compose.
  • SPA React + Redux Toolkit consumiendo API REST.
  • Pipeline GitHub Actions ejecutando pruebas y cobertura.
Todo el código convive en un solo repositorio para facilitar CI/CD y documentación.

Backend & Testing

  • FastAPI con endpoints CRUD (GET, POST, PUT, DELETE).
  • TestClient de Starlette + Pytest para validar flujos felices y errores.
  • pytest-cov asegura cobertura > 80 %.
  • Dockerfile liviano + Compose para levantar API y DB.

Frontend SPA

  • Vite + React + TypeScript.
  • Redux Toolkit para estado global de tareas.
  • Axios como cliente HTTP con configuración `VITE_API_URL`.
  • Vitest + Testing Library + cobertura V8.

Pipeline CI/CD

  • Workflow `CI` en GitHub Actions.
  • Instala Python 3.11, dependencias y ejecuta Pytest.
  • PYTHONPATH configurado para importaciones estables.
  • Meta: 10 ejecuciones verdes registradas como evidencia.

Métricas de Calidad

  • Cobertura backend: > 80 % (pytest-cov).
  • Cobertura frontend: > 80 % (vitest --coverage).
  • Tiempo promedio de build: < 2 min.
  • Tareas funcionales validadas vía suite automatizada.

Riesgos & Mitigaciones

  • Dependencias OS específicas: Contenedores y requirements fijados.
  • Regresiones en frontend: Pruebas Vitest + Playwright sugeridas.
  • Cobertura insuficiente: Scripts dedicados `pytest --cov` / `npm run test:coverage`.
  • Base de datos: Configuración Docker Compose reproducible.

Roadmap Próxima Iteración

  • Persistencia real en PostgreSQL usando SQLAlchemy ORM.
  • Pruebas E2E con Playwright y datos seed.
  • Despliegue automatizado a ambiente de staging (Render/Fly.io).
  • Monitoreo de métricas (coverage badge, Sonar o Codecov).

Cierre

El proyecto demuestra la integración de todas las competencias del curso: codificación disciplinada, pruebas exhaustivas y automatización CI/CD.
  • Repositorio documentado y reproducible.
  • Pipeline automatizado en GitHub Actions.
  • Frontend y backend listos para demo en video.
