Proyecto Caso Testigo - CI/CD

Ingeniería de Software FESC Unidad 3 Semestre VI

Transformamos el gestor de tareas empresarial en un caso práctico que integra backend y frontend profesionales, automatización de pruebas y despliegues continuos con GitHub Actions.

Objetivos

  • Implementar CI/CD end-to-end en un solo repositorio.
  • Unir competencias de codificación, testing y documentación.
  • Presentar evidencias completas para la rúbrica institucional.

Stack Clave

  • FastAPI · PostgreSQL · Docker
  • React · Redux Toolkit · Vite
  • Pytest · Vitest · GitHub Actions

Arquitectura integral

Backend & Datos

  • FastAPI sobre Python 3.11.
  • Persistencia en memoria + conectividad opcional PostgreSQL.
  • Docker Compose para API + DB compartida.

Frontend & UX

  • SPA en React/Vite con TypeScript.
  • Redux Toolkit centraliza estado y efectos.
  • Axios configurable vía `VITE_API_URL`.

Todo el código convive en un único repositorio, facilitando el versionamiento y la automatización de pruebas.

Backend & Testing

Características

  • Endpoints CRUD completos (`/tareas`).
  • Nuevo `/health` para monitoreo.
  • Dockerfile slim y variables por entorno.

Pruebas

  • Pytest + TestClient validando flujos felices y 404.
  • Fixture resetea la base en memoria.
  • `pytest --cov=app --cov-report=term-missing` > 80 %.

Frontend SPA

Stack

  • Vite + React 18 + TypeScript.
  • Redux Toolkit + hooks tipados.
  • Componentes modulares (Header, Resumen, Form, Lista).

Testing & UX

  • Vitest + Testing Library + cobertura V8 (`npm run test:coverage`).
  • Estado sincronizado con la API.
  • Diseño responsive listo para la demo.

Pipeline CI/CD

Workflow `CI` en GitHub Actions que instala Python 3.11, levanta el entorno virtual, instala dependencias y ejecuta Pytest automáticamente.

  • Variables configuradas para asegurar imports (`PYTHONPATH=${'{'}{'}'} github.workspace {'{'}{'}'}`).
  • Historial esperado: ≥10 builds verdes antes de la entrega.
  • Facilidad para extender con coverage badges o despliegue a staging.

Métricas de calidad

>80 % Cobertura backend (`pytest-cov`).
>80 % Cobertura frontend (`vitest --coverage`).
<2 min Tiempo promedio por build.
10+ Builds verdes registrados.

Riesgos & mitigaciones

Dependencias

Versiones fijadas y uso de contenedores para ambientes reproducibles.

Regresiones UI

Suite de Vitest y recomendación de Playwright E2E.

Cobertura

Scripts dedicados (`pytest --cov`, `npm run test:coverage`).

Base de datos

Docker Compose estandariza credenciales y puertos.

Roadmap próxima iteración

  • Persistencia real en PostgreSQL usando SQLAlchemy ORM.
  • Pruebas E2E con Playwright y datos sembrados.
  • Despliegue automatizado a un entorno cloud (Render/Fly.io).
  • Monitoreo de métricas (coverage badge, Codecov/SonarQube).
